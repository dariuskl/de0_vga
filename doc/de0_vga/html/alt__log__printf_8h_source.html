00001 /*  alt_log_printf.h
00002  *
00003  *  ALT_LOG is designed to provide extra logging/debugging messages from HAL
00004  *  through a different port than stdout.  It is enabled by the ALT_LOG_ENABLE
00005  *  define, which needs to supplied at compile time.  When logging is turned off, 
00006  *  code size is unaffected.  Thus, this should be transparent to the user
00007  *  when it is not actively turned on, and should not affect projects in any way.
00008  *
00009  *  There are macros sprinkled within different components, such as the jtag uart 
00010  *  and timer, in the HAL code.  They are always named ALT_LOG_<name>, and can be
00011  *  safely ignored if ALT_LOG is turned off. 
00012  *
00013  *  To turn on ALT_LOG, ALT_LOG_ENABLE must be defined, and ALT_LOG_PORT_TYPE and
00014  *  ALT_LOG_PORT_BASE must be set in system.h.  This is done through editing 
00015  *  <project>.ptf, by editing the alt_log_port_type & alt_log_port_base settings.
00016  *  See the documentation html file for examples.
00017  *
00018  *  When it is turned on, it will output extra HAL messages to a port specified
00019  *  in system.h.  This can be a UART or JTAG UART port.  By default it will 
00020  *  output boot messages, detailing every step of the boot process.  
00021  *
00022  *  Extra logging is designed to be enabled by flags, which are defined in 
00023  *  alt_log_printf.c.  The default value is that all flags are off, so only the
00024  *  boot up logging messages show up.  ALT_LOG_FLAGS can be set to enable certain
00025  *  groupings of flags, and that grouping is done in this file.  Each flag can 
00026  *  also be overridden with a -D at compile time.
00027  *
00028  *  This header file includes the necessary prototypes for using the alt_log
00029  *  functions.  It also contains all the macros that are used to remove the code
00030  *  from alt log is turned off.  Also, the macros in other HAL files are defined 
00031  *  here at the bottom.  These macros all call some C function that is in 
00032  *  alt_log_printf.c.
00033  *
00034  *  The logging has functions for printing in C (ALT_LOG_PRINTF) and in assembly
00035  *  (ALT_LOG_PUTS).  This was needed because the assembly printing occurs before
00036  *  the device is initialized.  The assembly function corrupts register R4-R7, 
00037  *  which are not used in the normal boot process.  For this reason, do not call
00038  *  the assembly function in C.
00039  *
00040  *  author: gkwan 
00041  */
